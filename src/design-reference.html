1. Project Overview & User Flow

Application Purpose

Desktop-first business portal for creating and managing Delivery Orders (DO), Purchase Orders (PO), and Invoices with PDF generation capabilities.

Complete User Flow

Login
→
Signup
(2 Steps)
→
Dashboard
↓
Create
DO/PO/Invoice
→
Success Page
+ Download PDF

Key Features

  • ✅ Two-step signup with company details collection
  • ✅ Company details auto-saved and reused in all documents
  • ✅ Persistent sidebar navigation (256px width)
  • ✅ Success page with PDF download after document creation
  • ✅ LocalStorage data persistence
  • ✅ Desktop-optimized layout (not mobile responsive)

Screen Inventory

Authentication (2)
Login & Signup with Company Details
Main Pages (2)
Dashboard & Company Settings
Document Creation (4)
Create DO, PO, Invoice + Success Page
List Views (3)
DO List, PO List, Invoice List

2. Authentication Screens

2.1 Login Screen

Layout: Full-screen gradient background (blue-50 to indigo-100), centered card
Card Width: 448px (max-w-md)
Card Style: White background, rounded-lg (8px), shadow-lg
Elements:
  • • Company icon: 64x64px, blue-600 background, rounded-xl
  • • Title: "Welcome Back" - centered
  • • Subtitle: "Sign in to your business portal" - gray-600
  • • Email input field
  • • Password input field
  • • Primary button: "Sign In" - full width, blue-600
  • • Toggle link: "Don't have an account? Sign up" - blue-600

2.2 Signup - Step 1 (User Info)

Layout: Same gradient background, same centered card
Fields:
  • • Full Name (required)
  • • Email (required)
  • • Password (required)
Button: "Continue to Company Details" - full width
Progress: "Step 1 of 2" shown in subtitle

2.3 Signup - Step 2 (Company Details)

Layout: Wider card - 768px (max-w-2xl)
Title: "Company Details"
Subtitle: "Tell us about your company - Step 2 of 2"
Form Grid: 2-column layout (grid-cols-2)
Fields:
  • • Company Name (full width, col-span-2)
  • • Street Address (full width, col-span-2)
  • • City, State (2 columns)
  • • ZIP Code, Country (2 columns)

Buttons: Back (outlined) + Complete Registration (primary) - flex layout

4. Document Creation Forms

4.1 Create Delivery Order (DO)

Layout: Multi-card layout, max-width 1280px (max-w-5xl)
Card 1 - DO Information:
  • • DO Number (auto-generated, editable)
  • • Date (date picker)
  • • Grid: 2 columns
Card 2 - Customer Information:
  • • Customer Name
  • • Customer Address (textarea, 3 rows)
Card 3 - Items:
  • • Dynamic rows with Description, Quantity, Unit
  • • Add/Remove buttons (Plus/Trash2 icons)
  • • "Add Item" button at bottom
Card 4 - Notes:
  • • Textarea (4 rows)
Actions: Create DO (primary) + Cancel (secondary)

4.2 Create Purchase Order (PO)

Similar to DO but with:
  • • Vendor Information instead of Customer
  • • Items include: Description, Qty, Unit, Unit Price, Total
  • • Auto-calculation of item totals
  • • Total Amount displayed at bottom of items

4.3 Create Invoice

Extended PO format with:
  • • Invoice Number, Date, Due Date (3 columns)
  • • Customer Information
  • • Items with pricing
  • • Tax calculation (editable tax rate %)
  • • Summary: Subtotal, Tax, Total Amount

4.4 Document Success Page

Layout: Full screen gray-50 background, centered card
Card Width: 768px (max-w-2xl)
Elements:
  • • Success icon: 80x80px, green-100 background, green-600 icon
  • • Title: "[Document Type] Created!" in green-900
  • • Subtitle: Document number and success message
  • • Summary box: gray-50 background with key details
  • • Two action buttons (grid-cols-2):
  •   - Download PDF (primary, blue-600)
  •   - View All [Documents] (secondary, outlined)
  • • Back to Dashboard (ghost button)

7.2 Typography

Font Family: System font stack (from globals.css)
Scale: Defined in globals.css - DO NOT override with Tailwind classes
Text Colors:
  • • Primary: gray-900 (#111827)
  • • Secondary: gray-600 (#4b5563)
  • • Tertiary: gray-500 (#6b7280)
  • • Disabled: gray-400 (#9ca3af)
Usage:
  • • h1: Page titles
  • • h2: Section titles
  • • h3: Card titles
  • • p: Body text
  • • .text-sm: Secondary info

7.3 Spacing

Tailwind Scale (px):
  • • 1 = 4px
  • • 2 = 8px
  • • 3 = 12px
  • • 4 = 16px
  • • 6 = 24px (common for cards)
  • • 8 = 32px (common for page padding)
  • • 12 = 48px
Common Patterns:
  • • Page padding: p-8
  • • Card padding: p-6
  • • Section spacing: space-y-6
  • • Input padding: p-3
  • • Button padding: py-3 px-6

7.4 Border Radius

  • • rounded-md (6px): Inputs, buttons
  • • rounded-lg (8px): Cards, containers
  • • rounded-xl (12px): Logo backgrounds
  • • rounded-full: Badges, avatars

7.5 Shadows

  • • shadow-sm: Subtle elevation
  • • shadow: Default card shadow
  • • shadow-lg: Modals, success cards

🎨 Design Notes for Figma

  • Create components for: Button, Input, Card, Badge, Nav Item, Table Row
  • Use Auto Layout extensively for responsive spacing
  • Create color styles for all grays, blues, and semantic colors
  • Create text styles for h1, h2, h3, body, and small text
  • Use consistent 8px grid system (spacing in multiples of 4px/8px)
  • Create variants for button states (hover, active, disabled)
  • Use frames for each screen (1440px width recommended)
  • Group sidebar as a fixed component across all screens
  • Add prototyping links to show complete user flow
